# The Lab - Environments & Infrastructure Breakdown

This document provides an overview of the different environments and infrastructure components used in The Lab homelab project, and how they are structured within the Terraform codebase.

## Overview

- The Lab runs a single k3s cluster spanning 3 on-premises nodes (Raspberry Pi & Samsung Laptop) with OCI cloud instances joined as additional worker nodes, annotated as `cloud-nodes`.
- All cluster resources are managed using Infrastructure as Code (IaC) principles, primarily utilizing Terraform.
- The cluster is provisioned using the k3s script.
- Each environment has its own dedicated branch in the Git repository to isolate changes and configurations.

## Environment Setup

| Node Group    | Hosted on                                               | Branch                              | Tools            | Infrastructure as Code | Cluster Provisioning      |
| ------------- | ------------------------------------------------------- | ----------------------------------- | ---------------- | ---------------------- | ------------------------- |
| On-Prem Nodes | Raspberry Pi, Samsung and Acer Laptop (3 nodes)         | `management`, `main`, `dev-lab`     | k3s + containerd | Terraform              | Ansible                   |
| Cloud Nodes   | OCI Instances (annotated as `cloud-nodes`)              | `management`, `main`, `dev-lab`     | k3s + containerd | Terraform              | Ansible                   |

## Cluster Resources Breakdown

| Namespace    | Workloads                                                                                                               |
| ------------ | ----------------------------------------------------------------------------------------------------------------------- |
| `management` | Jenkins, Harbor, Traefik, CertManager, ArgoCD, Longhorn, Nginx Proxy Manager, HashiCorp Vault, CrowdSec                 |
| `monitoring` | Kube-Prometheus Stack, FluentBit, OpenSearch, OpenSearch Dashboard, ntopng                                          |
| `prod-lab`   | AdGuard Home, Homepage (Dashboard), Actual Budget, Outline (Docs), Nextcloud, n8n, Arr Suite + qBittorrent, Minecraft Server |
| `dev-lab`    | Homelab tools under testing / experimentation                                                                           |

> **Note**: OneUpTime or Uptime Kuma will be deployed under the `monitoring` namespace for uptime monitoring, leveraging the cloud-nodes for external reachability.

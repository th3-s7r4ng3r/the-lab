# Module Breakdown

This document provides a detailed breakdown of the various terraform modules in the project, explaining their purpose and functionality.

    - cluster-templates
        - application
            - application.tf

            - deployment.tf

            - pv-claim.tf

        - dns-record
            - main.tf
            - secrets.tf

            - variables.tf

## Environments

Each environment (e.g., prod-lab, dev-lab) has its own set of terraform files that define the specific resources and configurations for that environment. These files utilize the templates defined in the `cluster-templates` module to create the necessary Kubernetes resources.

    - prod-lab
        - applications.tf: Define ArgoCD applications for the environment.
        - deployments.tf: Define custom kubernetes deployments for the cluster.
        - services.tf: Define kubernetes services for the cluster.
        - networks.tf: Define kubernetes namespaces and network policies.
        - dns-records.tf: Define DNS records for the environment for Nginx Proxy Manager and Cloudflare.
        - variables.tf: Define input variables for the environment.
        - pv-claims.tf: Define persistent volume claims for the resources in the environment.
        - main.tf: Main entry point for the environment. Define provider configurations and module calls here.

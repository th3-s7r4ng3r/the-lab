# Setting Up Kube-Prometheus-Operator

This guide provides step-by-step instructions on how to set up the Kube-Prometheus-Operator for monitoring the homelab k3s cluster in The Lab project.

## Important Points

- Kube-Prometheus-Operator with Grafana is deployed into the `monitoring` namespace.
- This is deployed using Helm charts managed via Terraform.
- A single Prometheus instance scrapes metrics from all relevant namespaces across the cluster.

## Steps to Set Up Kube-Prometheus-Operator

1. **Create Kube-Prometheus-Operator Deployment using Terraform**:
   - Switch to the `management` branch in the `the-lab` repository.
   - Create a new Terraform module in the `deployments.tf` file in `k3s-cluster/monitoring/`.
   - Define the Kube-Prometheus Stack Helm chart parameters such as release name, chart version, namespace, and values overrides. (Refer to existing configurations in `k3s-cluster/monitoring/deployments.tf`.)
   - Further customize the values for Grafana and Prometheus as in the `variables.tf` file in the same directory.
   - Update the `secrets.tf` file to include any necessary credentials for Grafana (e.g., admin password), referencing HashiCorp Vault.
   - Add the `monitoring` label to the namespaces with the value defined in the Prometheus Scrape Selectors in the `variables.tf` file. This is to ensure that Prometheus scrapes metrics from the namespaces.
2. **Create DNS Records**:
   - Open the `dns-records.tf` file in `k3s-cluster/monitoring/`.
   - Add DNS records for Grafana by referring to existing records.
   - Commit the changes into the relevant branch to deploy automatically via Terraform.
3. **Access Grafana UI**:
   - Open your web browser and navigate to the Grafana UI at `https://grafana.<homelab-domain>`.
   - Log in using the admin credentials stored in HashiCorp Vault.
4. **Verify Deployment**:
   - In the Grafana UI, navigate to the Data Sources section.
   - Ensure that Prometheus is listed as a data source and is functioning correctly.
   - Head to the Dashboards section and verify that the default dashboards are available and displaying data.
5. **Configure Alerting Rules**:
   - Navigate to the Alerting section in Grafana. Many default alerting rules are pre-configured.
   - Customize or add new alerting rules as per the homelab's requirements.

# Setting Up Argo CD Application

This document provides a step-by-step guide on how to set up an Argo CD application for deploying and managing applications in The Lab homelab project.

## Important Points

- Argo CD is deployed into the `management` namespace using Helm charts.
- Argo CD syncs manifests directly from the `the-lab` Git repository.
- Each namespace (`management`, `monitoring`, `prod-lab`, `dev-lab`) can have its own Argo CD application defined in the respective `applications.tf` file under `k3s-cluster/`.

## Steps to Set Up Argo CD Application

1. **Create Application using Terraform**:
   - Create or update the `applications.tf` file under the relevant namespace directory (e.g., `k3s-cluster/prod-lab/applications.tf`) in the `the-lab` repository.
   - Define the Argo CD application required parameters such as application name, source repository, path, and destination namespace. (Refer to existing `applications.tf` files for structure and values.)
   - Commit the changes into the relevant branch to deploy automatically via Terraform.
2. **Access Argo CD UI**:
   - Open your web browser and navigate to the Argo CD UI at `https://argocd.<homelab-domain>`.
   - Log in using the admin credentials stored in HashiCorp Vault.
3. **Verify Application Deployment**:
   - In the Argo CD UI, navigate to the Applications section.
   - Locate the newly created application.
   - Check the sync status and health of the application to ensure it has been deployed successfully.
4. **Verify Sync and Health**:
   - After making changes, ensure that the application is in sync and healthy by checking the Argo CD UI.
   - If there are any issues, refer to the logs and troubleshoot accordingly.
5. **Setup GitHub Webhooks**:
   - Create a GitHub webhook in the `the-lab` repository with the following settings:
     - Payload URL: `https://argocd.<homelab-domain>/api/webhook`
     - Content type: `application/json`
     - Secret: Use the existing webhook secret defined in the Argo CD settings. (if not generate new secret and add it `kubectl edit secret argocd-secret -n management` as `webhook.github.secret: "base64-encoded-secret"`)
     - Events: Select "Just the push event".
   - This will ensure that any changes pushed to the repository will automatically trigger a sync in Argo CD.

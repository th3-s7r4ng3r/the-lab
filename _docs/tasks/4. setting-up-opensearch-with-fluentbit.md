# Setting up OpenSearch with Fluent Bit For Log Management

This document provides a step-by-step guide on how to set up OpenSearch with Fluent Bit for log management in The Lab homelab project.

## Important Points

- OpenSearch and OpenSearch Dashboards are deployed in the `monitoring` namespace using Helm charts.
- Fluent Bit is used as the log forwarder to collect logs from pods and send them to OpenSearch. It is deployed using Helm charts in the `monitoring` namespace.
- Logs are indexed in OpenSearch with different indices for each namespace.

## Steps to Set Up OpenSearch with Fluent Bit

1. **Deploy OpenSearch and OpenSearch Dashboards using Helm**:
   - Refer to existing Helm chart configurations in `k3s-cluster/monitoring/deployments.tf`.
   - Ensure to set the necessary parameters (cluster name, namespace, resource limits, storage class, etc.) in the Helm values.
   - Commit the changes into the relevant branch to deploy automatically via Terraform.
2. **Deploy Fluent Bit using Helm**:
   - Switch to the `management` branch first. Then create a new Helm release for Fluent Bit in the `k3s-cluster/monitoring/deployments.tf` file.
   - Configure Fluent Bit to collect logs from required namespaces and forward them to the OpenSearch cluster. (Refer to the configurations in `k3s-cluster/monitoring/variables.tf` for necessary Fluent Bit configs such as service, input, and filters.)
3. **Configure Fluent Bit to Send Logs to OpenSearch**:
   - In the Fluent Bit Helm values, set up the output configuration to point to the OpenSearch cluster endpoint. (Refer to `k3s-cluster/monitoring/secrets.tf` for sample configurations.)
4. **Store Secrets in Vault**:
   - After updating the `secrets.tf` files, ensure any sensitive values are stored in HashiCorp Vault and referenced accordingly in the Terraform configs.
5. **Create DNS Records**:
   - Open the `dns-records.tf` file in `k3s-cluster/monitoring/`.
   - Add DNS records for OpenSearch Dashboards by referring to existing records.
   - Commit the changes into the relevant branch to deploy automatically via Terraform.
6. **Access OpenSearch Dashboards**:
   - Open your web browser and navigate to the OpenSearch Dashboards UI at `https://opensearch.<homelab-domain>`.
   - Log in using the admin credentials stored in HashiCorp Vault.
7. **Create Index Patterns**:
   - In the OpenSearch Dashboards UI, navigate to the "Index Patterns" section.
   - Create index patterns for each namespace to visualize logs. For example, create an index pattern like `logs-prod-lab-*` for the prod-lab namespace.
8. **Verify Log Ingestion**:
   - In the OpenSearch Dashboards UI, navigate to the "Discover" section.
   - Select the appropriate index pattern and verify that logs from the configured namespaces are being ingested and displayed correctly.
9. **Set Up Alerts and Visualizations**:
   - Use OpenSearch Dashboards to create visualizations and dashboards for log data.
   - Set up alerts based on specific log patterns or thresholds as needed.
